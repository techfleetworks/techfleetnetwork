External audit (2026-04-18) recommended 9 changes. All applied:

1. **CSP** — Added `<meta http-equiv="Content-Security-Policy">` to `index.html` so the production Cloudflare host (which doesn't honour `public/_headers`) gets a real CSP. `_headers` retained as defense in depth and updated to allow gtag + cookieyes.
2. **Pinned dependencies** — Stripped every `^` and `~` in `package.json`. `vaul` was specifically called out as low-maintenance / supply-chain risk.
3. **Two-step fanout** — `notify_project_opening` now drops a single `notification_fanout_jobs` row instead of looping over recipients in the projects-UPDATE transaction. New SECURITY DEFINER `process_notification_fanout_chunk(job_id, chunk_size)` drains in chunks of 500. New `supabase/functions/process-notification-fanout/` worker calls the RPC; schedule via pg_cron or admin trigger.
4. **Fleety KB cache** — Module-level cache in `supabase/functions/techfleet-chat/index.ts` (`KB_CACHE_TTL_MS` = 5 min) with in-flight coalescing.
5. **Dropped announcements realtime** — `useAnnouncementRealtime` is now a no-op. The 60s `useLatestAnnouncements` poll is the sole delivery channel. Removes 10k WebSocket frames per admin INSERT.
6. **Dashboard single-round-trip RPC** — `get_dashboard_overview(uuid)` returns phase counts + general app + project apps in one call. Hook lives at `src/hooks/use-dashboard-overview.ts`.
7. **Rate-limit RPC** — `RateLimitService.check` now calls `check_rate_limit` directly via PostgREST RPC instead of the `rate-limit` edge function (zero cold start).
8. **TS pin / test alignment** — `RateLimitService` still fails open and tests already mock at the service boundary, so no test changes required.

Operational notes:
- The fanout worker requires admin JWT or service-role auth.
- If realtime announcements are ever needed again, add jitter and scope the subscription to /updates + /dashboard only.
- Bumping any dep now requires an explicit version bump (no auto-minor updates).
