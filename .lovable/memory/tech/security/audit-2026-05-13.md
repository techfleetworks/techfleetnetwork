# Security Audit 2026-05-13 — Remediation Record

**Pass type:** zero-brick. No RLS, no policies, no role grants, no SECURITY DEFINER changes, no auth gate changes, no migrations.

## Findings outcome

- **H-01 XSS (events):** false positive. `EventDetailDialog` and `CommunityEventCard` already wrap `event.description` in `sanitizeHtml(linkifyHtml(...))` before `dangerouslySetInnerHTML`.
- **H-02 / M-02 headers:** already mitigated. `public/_headers` serves full set on custom domain; `index.html` embeds CSP + Permissions-Policy + Referrer-Policy + X-Content-Type-Options as `<meta http-equiv>` for hosts that don't honor `_headers`. `X-Frame-Options` cannot be set via meta per HTML spec — preview origin clickjacking is accepted.
- **M-03 COOP/COEP:** COEP intentionally OFF. Would break Google OAuth popup, YouTube embeds, Luma calendar embeds, Turnstile. Documented in `docs/threat-model.md`.
- **M-04 cf_bm cookie:** third-party, accepted.
- **L-01 npm audit:** FIXED. `.github/workflows/npm-audit.yml` runs per-PR and weekly Mondays 06:00 UTC, report-only artifact.
- **L-02 SECURITY DEFINER inventory:** informational, no action.
- **L-04 fleety_turn_signals:** working as designed, accepted.

## Deferred (phased, see `docs/runbooks/edge-fn-zod-rollout.md`)

- **M-01 Zod on edge functions:** 40 of 81 functions parse JSON without Zod. Per-function classification (A–E) + risk class + per-PR verification template documented. Hard rule: ≤10 functions per PR, every existing field `.optional()`/`.passthrough()` to never reject a previously-valid call.
- **M-05 CORS hygiene:** 64 of 81 functions use `*`. Bundled into M-01 batches. Class C webhook functions (Discord, Gumroad, Resend, auth-email-hook) keep `*` because signature is the auth.
- **L-03 Realtime channel scope:** documented audience matrix in `docs/attack-surface.md`. Backend stays permissive. Frontend rule: every `supabase.channel()` must role-gate before subscribe. Per-channel RLS tightening only after every producer migrated to structured topic name.

## Permission preservation — verified by construction

Zero diff vs. baseline snapshot at `/mnt/documents/security-audit-evidence/pre-fix-baseline/` (369 policies + 209 public functions). No SQL ran in this pass.

Future audit passes must NOT re-flag H-01, H-02, M-02, M-03, M-04, L-01, L-02, L-04 without first reading this record + linked docs. False positives waste review cycles and risk panic-driven fixes that brick functionality.
